## Images temporaires à remplacer

Les fichiers SVG sont des visuels provisoires. Conserve les mêmes noms lors du remplacement pour ne pas modifier le HTML.

- `assets/images/projects/sanitoral.svg` : remplacer par une capture de la page principale de ton tableau de bord Sanitoral, montrant les KPI, l’avancement, les retards et les budgets.
- `assets/images/projects/uoi-games.svg` : remplacer par une capture de ton tableau de bord, de l’analyse du questionnaire ou de la diapositive présentant la recommandation ASHFALL.
- `assets/images/projects/bottleneck-python.svg` : remplacer par une capture propre de ton notebook Python avec un graphique de stock, un boxplot ou une analyse du chiffre d’affaires.
- `assets/images/projects/bottleneck-power-bi.svg` : remplacer par une capture du tableau de bord Power BI Bottleneck avec les KPI de ventes, de marge et de stock.
- `assets/images/projects/retailinsight360.svg` : remplacer par une capture du schéma relationnel, des résultats SQL ou de ta présentation NPS pour BestMarket.
- `assets/images/projects/immobilier-python.svg` : remplacer par une capture du notebook avec le modèle de régression, la courbe des prix ou la visualisation des clusters immobiliers.
- `assets/images/projects/aeroworld.svg` : remplacer par une capture de ton portfolio final, de ton diagramme de Gantt ou de ton tableau de bord de présentation du profil.
- `assets/images/projects/primero-bank.svg` : remplacer par une capture de ta présentation Primero Bank montrant une visualisation du churn ou le profil des clients à risque.
- `assets/images/projects/premieres-requetes-sql.svg` : remplacer par une capture de ton schéma relationnel ou d’une requête SQL avec son résultat dans SQLiteStudio.
- `assets/images/projects/dev-immediat-rgpd.svg` : remplacer par une capture des étapes Power Query d’anonymisation ou un aperçu du fichier CRM anonymisé sans donnée personnelle.

Dimensions recommandées :

- largeur : 1 280 px ;
- hauteur : 800 px ;
- ratio : 16:10 ;
- format recommandé : WebP ou PNG.

Aucun nouveau JavaScript n’a été nécessaire pour la page Projets.

## Publication sur GitHub Pages

1. Décompresse l’archive.
2. Dépose tout le contenu à la racine de ton dépôt GitHub.
3. Dans `Settings` puis `Pages`, choisis `Deploy from a branch`.
4. Sélectionne la branche `main` et le dossier `/ (root)`.
5. Enregistre.
